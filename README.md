# LLM Software Aging in Flask Applications

This repository contains the artifacts, scripts, and experimental setup used in the study of **software aging in LLM-generated Python/Flask web applications**.

The project investigates whether web applications automatically generated by Large Language Models (LLMs) exhibit signs of software aging during long-running execution, such as progressive memory consumption and response-time degradation.

## Objective

The main goal of this project is to analyze whether LLM-generated systems are also subject to software aging effects commonly observed in traditionally developed software.

More specifically, the study evaluates:
- progressive changes in **memory usage**
- variations in **response time**
- behavior of **CPU usage**
- changes in the **number of running processes**

## Research Context

Recent studies on LLM-generated code have mainly focused on aspects such as correctness, quality, security, and efficiency. However, there is still limited evidence on whether applications generated by LLMs are prone to **software aging symptoms** under prolonged execution.

To investigate this gap, this repository provides the material used to execute controlled long-running experiments with generated web applications.

## Experimental Design

The experiments were conducted using Python/Flask web applications generated from natural language prompts.

Each application was executed continuously for **48 hours**, while metrics related to performance and resource usage were collected.

### Monitored metrics
- Memory consumption
- CPU utilization
- Number of processes
- Request response time

### Statistical analysis
To identify trends in the collected time series, the study applies:
- **Mann-Kendall trend test**
- **Sen’s slope estimator**

These methods are used to verify whether there is statistically significant progressive degradation over time.
